autofill credential harvesting detection — kuosmanen 2017 + tóth DEF CON 33 2025

safe simulation. This page demonstrates AutofillGuard detection capabilities. No real credentials are collected or transmitted. All attack scenarios run in isolated sandboxes within this page. The extension's content script detects the hidden fields and fires alerts.

scenario a: hidden field harvest (kuosmanen 2017)

A visible email field is shown to the user, but a display:none password field silently captures autofilled credentials. The form submits both fields to an external URL. This is the original 2017 autofill phishing technique.

visible email field → hidden password field (display:none) → cross-origin form action → credentials exfiltrated

scenario b: extension clickjacking (tóth 2025)

The page sets opacity:0 on a container element, making everything inside invisible — including the password manager's autofill popup. The user sees an innocuous overlay and unknowingly clicks the invisible autofill button. This is the DEF CON 33 DOM-based Extension Clickjacking vector.

container opacity: 0 → password field inside (invisible) → PM autofill UI invisible → user clicks invisible autofill

scenario c: off-screen field + form exfiltration

A password field is positioned at left:-9999px (off-screen) while a visible decoy form is shown. The off-screen field captures autofilled credentials and the form submits them to an external endpoint.

visible decoy form → off-screen password field (left:-9999px) → external form action → silent exfiltration
