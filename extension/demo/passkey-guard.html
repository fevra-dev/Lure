passkey credential substitution — webauthn api interception

This demo simulates WebAuthn/Passkey credential substitution attacks by calling the signal analysis functions directly. No real passkeys are created or modified. The PasskeyGuard analyzes credential requests for suspicious relying party IDs.

attack chain: passkey sync fabric phishing

Attackers intercept navigator.credentials.create() to redirect passkey registration to an attacker-controlled relying party. Because passkeys sync across devices (Google Password Manager, iCloud Keychain), compromising one passkey gives access to ALL synced credentials. No browser or competitor tool currently defends against this. DEF CON 33 (Spensky) + IEEE S&P 2026 (SquareX) confirmed the attack.
